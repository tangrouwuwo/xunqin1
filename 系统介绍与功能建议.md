# 寻亲网站系统介绍

## 一、系统概述

寻亲网站是一个面向社会的**公共福利寻亲平台**，致力于帮助失散家庭重新团聚。系统采用前后端分离架构，通过互联网技术手段连接寻亲者、志愿者、线索提供者和管理员，构建了一个多方协作的寻亲生态体系。

**系统使命**：让爱回家，让亲情重聚

**核心价值**：
- 为失散家庭提供便捷的寻亲信息发布与查询渠道
- 为志愿者提供参与寻亲公益活动的平台
- 为线索提供者提供快捷的线索上报途径
- 通过AI技术辅助寻亲信息的智能分析与匹配

## 五、角色体系

系统设计了**四种角色**，各角色职责与权限如下：

### 1. 管理员 (ADMIN)
- 用户管理：创建、编辑、禁用系统用户
- 寻亲信息管理：审核、审批、变更加密寻亲信息
- 线索管理：查看所有线索，指派给志愿者核实
- 成功案例管理：审核发布成功案例
- 任务管理：创建任务并指派给志愿者
- 志愿活动管理：创建和管理志愿活动
- 社区管理：审核和管理社区帖子

### 2. 寻亲者 (SEEKER)
- 发布寻亲信息：创建和管理自己的寻亲信息
- 查看相关线索：查看与自己寻亲信息相关的线索
- 任务管理：查看系统分配的任务
- 社区交流：参与社区讨论
- 消息通知：接收线索和任务通知

### 3. 志愿者 (VOLUNTEER)
- 线索核实：接收并核实线索
- 任务接取：从任务大厅接取任务
- 志愿活动：报名参加志愿活动
- 社区交流：参与社区讨论

### 4. 线索提供者 (CLUE_PROVIDER)
- 提交线索：提交流失人员相关线索
- 线索管理：查看自己提交的线索状态
- 社区交流：参与社区讨论

## 六、功能模块详解

### 6.1 公共功能

| 功能 | 描述 |

| **寻亲信息查询** | 按省份、性别、年龄段、关键词搜索寻亲信息 |
| **寻亲信息详情** | 查看寻亲信息的详细内容、照片、线索展示 |
| **成功案例** | 展示已团聚的成功案例，包含故事和照片 |
| **社群交流** | 公开社区，寻亲者、志愿者、线索提供者均可参与 |
| **志愿者招募** | 展示志愿者价值和近期活动，引导用户注册成为志愿者 |
| **用户注册/登录** | 支持多种角色注册，JWT令牌认证 |

### 6.2 认证与安全

| 功能 | 描述 |
|------|------|
| **JWT令牌认证** | 无状态认证，支持令牌刷新 |
| **防缓存攻击** | 通过Cache-Control头、pageshow事件、meta标签三重防护，防止浏览器后退缓存导致的未授权访问 |
| **防密码自动填充** | 使用隐藏假表单、动态字段名、输入框类型延迟切换等策略 |
| **角色路由守卫** | 前端路由按角色权限控制访问，未登录重定向至首页 |

### 6.3 管理员后台

| 功能 | 描述 |
|------|------|
| **仪表盘** | 系统数据概览和统计 |

## 建议一：人脸识别比对系统

### 功能描述
接入人脸识别AI引擎，允许用户上传失踪人员照片和疑似人员照片，系统自动进行人脸相似度比对，给出匹配度评分。管理员和寻亲者可以通过人脸搜索在海量寻亲信息中快速匹配相似人员。

### 具体实现
- **人脸特征提取**：对上传照片进行人脸检测和特征向量提取
- **1:1 比对**：两张照片直接比对，输出相似度分数
- **1:N 搜索**：一张照片在数据库中搜索最相似的N个结果
- **年龄增长模拟**：AI模拟失踪人员多年后的外貌变化
- **模糊照片增强**：对模糊、老照片进行AI画质增强

### 价值
大幅提高寻亲匹配效率，将传统的人工翻找改为智能识别匹配。

## 建议十三：寻亲信息自动匹配与推荐

### 功能描述
基于机器学习算法，对全平台的寻亲信息和线索进行智能匹配分析。当新的寻亲信息或线索发布时，系统自动与现有数据进行交叉比对，向相关用户推送匹配提醒。

### 具体实现
- **特征向量化**：将姓名、年龄、身高、体貌特征、失踪地点等关键信息转化为特征向量

- **匹配评分**：对每条匹配结果给出可信度评分，优先展示高匹配度结果
- **主动通知**：匹配度超过阈值时，主动向相关用户发送通知

### 价值
变被动查询为主动推送，让寻亲信息和线索自动"碰撞"，大幅提高匹配效率。
